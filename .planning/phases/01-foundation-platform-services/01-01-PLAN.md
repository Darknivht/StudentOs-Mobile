# Plan 01-01: Monorepo Scaffold, Expo Project Setup, NativeWind v4 Config

## Goal

Establish the monorepo structure with `apps/native/` (Expo) and `packages/shared/` (@studentos/shared), configure NativeWind v4 with the web app's design tokens, and verify the app launches on Android.

## Requirements

- **FND-01**: App runs as native React Native (Expo) Android app
- **FND-02**: Monorepo structure with @studentos/shared package
- **FND-05**: NativeWind replicates existing design system tokens and dark mode

## Threat Model

| ID | Threat | Mitigation |
|----|--------|------------|
| T-1-01 | Auth tokens stored in plaintext in later phases | Shared package forces SecureStore injection — no localStorage default |
| T-1-02 | Hardcoded API keys persist | Env validation scaffolding established here, enforced in Plan 01-04 |

## Tasks

### Wave 1

- [ ] **01-01-01**: Initialize monorepo root package.json with npm workspaces
  - Create root `package.json` with `{ "workspaces": ["apps/*", "packages/*"], "private": true }`
  - Create `apps/` and `packages/` directories
  - Create root `tsconfig.json` with project references to each workspace
  - Create root `.gitignore` (update existing to include `node_modules`, `.expo/`, `dist/`)
  - **Verify**: `npm install` resolves workspaces without errors
  - **Req**: FND-02

- [ ] **01-01-02**: Create `@studentos/shared` package skeleton
  - Create `packages/shared/package.json` with name `@studentos/shared`, version `0.0.1`
  - Peer deps: `react`, `@supabase/supabase-js`, `@tanstack/react-query`
  - Create `packages/shared/tsconfig.json` extending root config
  - Create `packages/shared/src/index.ts` — initial exports (empty placeholder)
  - Create `packages/shared/src/lib/utils.ts` — copy `cn()` from `studentoss/src/lib/utils.ts`
  - Create `packages/shared/src/types/database.ts` — copy Supabase types from `studentoss/src/integrations/supabase/types.ts`
  - **Verify**: `import { cn } from '@studentos/shared'` resolves in consuming package
  - **Req**: FND-02

- [ ] **01-01-03**: Scaffold Expo project in `apps/native/`
  - Run `npx create-expo-app@latest apps/native --template blank-typescript`
  - Update `apps/native/package.json` to add dependency on `@studentos/shared`
  - Verify `app.json` has correct `name`, `slug`, `scheme` (`studentos`)
  - Set `jsEngine: "hermes"` in `app.json` (should be default for SDK 54+)
  - Add `"plugins": []` placeholder in `app.json` for future config plugins
  - **Verify**: `npx expo start` launches dev server without errors
  - **Req**: FND-01

- [ ] **01-01-04**: Configure Metro for monorepo
  - Create `apps/native/metro.config.js` using `@expo/metro-config` with monorepo support
  - Reference `../../packages/shared` in `watchFolders` (if not auto-configured)
  - **Verify**: Metro resolves `@studentos/shared` imports from `apps/native/`
  - **Req**: FND-02

- [ ] **01-01-05**: Install and configure NativeWind v4
  - Install: `nativewind@4`, `tailwindcss@3`, `nativewind/metro`
  - Create `apps/native/babel.config.js` with `nativewind/babel` plugin
  - Create `apps/native/global.css` with `@tailwind base; @tailwind components; @tailwind utilities;`
  - Update `apps/native/metro.config.js` with `nativewind/metro` integration
  - Create `apps/native/tailwind.config.ts` — port design tokens from `studentoss/tailwind.config.ts`:
    - Copy `theme.extend.colors` (primary, secondary, destructive, muted, background, foreground, etc.)
    - Copy `theme.extend.borderRadius`, `theme.extend.fontFamily`
    - Set `content` paths: `./app/**/*.{ts,tsx}`, `./components/**/*.{ts,tsx}`, `./features/**/*.{ts,tsx}`
    - Add `presets: ["nativewind/tailwind/css"]` (or equivalent NativeWind v4 preset)
  - Pin `nativewind@4` in `package.json` to prevent v5 upgrade: `"nativewind": "4.x"`
  - **Verify**: A `<View className="bg-primary p-4"><Text className="text-primary-foreground">Hello</Text></View>` renders with correct colors
  - **Req**: FND-05

- [ ] **01-01-06**: Create stub root layout that imports global.css
  - Create `apps/native/app/_layout.tsx` — minimal root layout:
    ```tsx
    import "../global.css";
    import { Stack } from "expo-router";

    export default function RootLayout() {
      return <Stack />;
    }
    ```
  - **Verify**: `npx expo start` + Android emulator shows blank app with no errors, NativeWind classes apply
  - **Req**: FND-01, FND-05

- [ ] **01-01-07**: Verify Android launch
  - Run `npx expo run:android` (or `npx expo start` + press `a`)
  - App launches on emulator, shows blank screen with no crash
  - Verify Hermes is active (check Metro logs for "HermesEngine")
  - **Verify**: App icon visible in launcher, app opens without crash
  - **Req**: FND-01

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Package manager | npm | Consistency with existing `studentoss/package.json` which uses npm |
| NativeWind version | v4 (NOT v5) | v5 is beta/preview; v4 is stable with TailwindCSS v3 |
| TailwindCSS version | v3 | Matches web app; v4 syntax not needed |
| JS engine | Hermes | Default in Expo SDK 54+, bytecode compilation, better perf on 2GB devices |
| Monorepo tool | npm workspaces | Simplest, no extra tooling, Expo has first-class support |

## Out of Scope

- Moving `studentoss/` to `apps/web/` (deferred — not needed for Phase 1 functionality)
- Deleting the Capacitor `android/` wrapper (deferred to avoid breaking web app until native works)
- `@studentos/shared` hooks (Phase 2+)
- Any feature screens (Phase 3+)
