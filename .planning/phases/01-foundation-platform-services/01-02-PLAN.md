# Plan 01-02: Expo Router File-Based Routing with (auth)/(tabs) Groups and Deep Linking

## Goal

Establish the complete Expo Router navigation structure with (auth) and (tabs) route groups, auth redirect logic, deep linking for password reset, and dark mode toggle — all feature screens are stubs that will be implemented in later phases.

## Requirements

- **FND-04**: Expo Router file-based routing replaces React Router
- **NAVL-01**: Bottom navigation with 5 tabs (Home, Study, Exams, Social, Profile)
- **NAVL-02**: AppLayout wraps authenticated routes with auth check and redirect
- **NAVL-03**: Deep linking for password reset magic links and shareable content URLs
- **NAVL-04**: Dark mode toggle matching web app behavior

## Threat Model

| ID | Threat | Mitigation |
|----|--------|------------|
| T-1-03 | Deep link hijacking — malicious app intercepts password reset | Use unique scheme `studentos://`, validate token format before processing |

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Tab count | 5 | NAVL-01 specifies Home, Study, Exams, Social, Profile |
| Auth redirect pattern | `useSegments()` + `useRouter()` | Expo Router recommended pattern for auth guards |
| Deep link scheme | `studentos://` | Short, unique, matches app slug |
| Icon library | `@roninoss/icons` (or `lucide-react-native`) | Expo-compatible icon library; lucide has RN package |
| Dark mode persistence | `expo-sqlite/kv-store` | Replaces `localStorage` for theme preference |
