# Plan 01-04: Cross-Cutting Platform Services (SSE, KaTeX, Env Validation, Error Boundaries, 2GB AVD Profiling)

## Goal

Implement the remaining cross-cutting platform services that all features depend on: SSE streaming client for AI responses, KaTeX math rendering test component, environment variable validation with zod, per-route error boundaries, 2GB AVD profiling setup, and bundle size baseline measurement.

## Requirements

- **FND-03**: Environment variables validated at startup with no hardcoded keys
- **FND-06**: Per-route error boundaries prevent one route crash from taking down the entire app
- **FND-07**: App performs smoothly on low-end Android devices (2GB RAM, Android 8+)
- **FND-08**: Bundle size optimized with code splitting, lazy loading, and tree shaking
- **(Indirectly validates)**: AITR-01 (SSE streaming), AITR-04 (KaTeX rendering) — test components only

## Threat Model

| ID | Threat | Mitigation |
|----|--------|------------|
| T-1-02 | Hardcoded Paystack test key `pk_test_58f4b0b6d69b01ee0e2b3a78b5de225b1ebd0ed8` in bundle | Zod env validation requires key from env var; no hardcoded fallback |
| T-1-04 | Missing env vars cause silent failures | Zod schema throws on parse failure; app shows env error screen instead of crashing |
| T-1-05 | Error boundaries leak stack traces in production | Sanitize error messages in production builds |

- [ ] **01-04-02**: Create SSE streaming client
  - Create `apps/native/services/sse-client.ts`:
    ```typescript
    export async function* streamSSE(
      url: string,
      options: { body: object; headers?: Record<string, string>; token?: string }
    ): AsyncGenerator<string, void, undefined> {
      const response = await fetch(url, {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          Authorization: options.token ? `Bearer ${options.token}` : '',
          ...options.headers,
        },
        body: JSON.stringify(options.body),
      });

      if (!response.ok) {
        throw new Error(`SSE request failed: ${response.status}`);
      }

      const reader = response.body?.getReader();
      if (!reader) throw new Error('No response body');

      const decoder = new TextDecoder();
      let buffer = '';

      while (true) {
        const { done, value } = await reader.read();
        if (done) break;

        buffer += decoder.decode(value, { stream: true });
        const lines = buffer.split('\n');
        buffer = lines.pop() || '';

        for (const line of lines) {
          if (line.startsWith('data: ')) {
            const data = line.slice(6);
            if (data === '[DONE]') return;
            yield data;
          }
        }
      }
    }
    ```
  - Create `apps/native/services/__tests__/sse-client.test.ts` — unit test with mock fetch
  - **Verify**: Unit test passes; SSE client parses data lines from ReadableStream chunks

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| SSE implementation | Custom `fetch` + `ReadableStream` | No external dependency, full control, Hermes supports ReadableStream |
| KaTeX rendering | `react-native-math-view` (primary), WebView fallback | Native rendering for inline math; WebView only for full documents |
| Env var prefix | `EXPO_PUBLIC_` | Expo convention for client-side env vars inlined by Metro |
| Error boundary depth | Root + per-route on key screens | Root catches everything; per-route on data-fetching screens |
| Production error display | Generic message only (no stack trace) | Prevents information leakage (T-1-05) |
| 2GB AVD API level | 28 (Android 9) | Covers Android 8+ target; Android 9 is stable and widely available |
