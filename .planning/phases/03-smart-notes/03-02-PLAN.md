2. Create src/services/notes/fileUploader.ts:

- Export function uploadFileToStorage(file: {uri: string, name: string, type: string}, userId: string): Promise of {url: string, path: string}
- Upload to Supabase Storage bucket 'notes-uploads' using supabase?.storage.from('notes-uploads').upload(path, blob, {contentType, upsert: false})
- Storage path format: userId/Date.now()\_sanitizedFileName
- Sanitize fileName: replace spaces and special chars with underscores
- After upload, get public URL: supabase?.storage.from('notes-uploads').getPublicUrl(path)
- Max file size: 10MB - reject larger files with Error('File too large. Maximum size is 10MB.')
- Read file as blob: const response = await fetch(file.uri); const blob = await response.blob();
- Return {url: publicUrl, path: storagePath}

3. Create src/services/notes/textExtractor.ts:

- Export function extractTextFromFile(fileUri: string, fileName: string, sourceType: 'pdf' | 'docx'): Promise of {text: string, sourceType: string, pageCount?: number}
- Read file as base64 using expo-file-system: FileSystem.readAsStringAsync(fileUri, {encoding: FileSystem.EncodingType.Base64})
- Call Supabase edge function: supabase?.functions.invoke('extract-pdf-text', {body: {file: base64Content, fileName, fileType: sourceType}})
- Parse response: data?.text or data?.content
- On network error: call addToOfflineQueue() and return {text: '', sourceType} (empty text indicates offline)
- Export function extractTextFromImage(imageUri: string): Promise of {text: string, sourceType: string}
- Read image as base64 using expo-file-system
- Call Supabase edge function: supabase?.functions.invoke('extract-pdf-text-ocr', {body: {image: base64Content}})
- On network error: queue offline, return empty text
- Export function addToOfflineQueue(item: {noteId: string, fileUri: string, fileName: string, sourceType: 'pdf' | 'docx' | 'image'}): void
- Store in localStorage under key 'offline_extraction_queue' as JSON array
- Generate unique ID: Date.now().toString(36) + Math.random().toString(36).slice(2)
- Export function processOfflineQueue(): Promise of void
- Called when app comes back online (use NetInfo listener)
- For each queued item: retry extraction, update note content via supabase
- Remove successfully processed items from queue
- Export function getOfflineQueue(): array of queued items

4. Create src/services/notes/index.ts:

- Re-export uploadFileToStorage from './fileUploader'
- Re-export extractTextFromFile, extractTextFromImage, addToOfflineQueue, processOfflineQueue, getOfflineQueue from './textExtractor'

5. Create src/hooks/useFileUpload.ts:

- Export function useFileUpload(noteId?: string)
- State: isUploading: boolean, isExtracting: boolean, uploadProgress: number (0-100), error: string | null, extractedText: string | null, fileUrl: string | null, sourceType: SourceType | null
- Import useNotes for updateNote
- Method uploadAndExtract(fileUri: string, fileName: string, fileType: SourceType, userId: string): Promise of void
  - Step 1: Set isUploading=true, uploadProgress=0
  - Step 2: Upload file to Supabase Storage via uploadFileToStorage - update uploadProgress to 50 on success
  - Step 3: Set isUploading=false, isExtracting=true
  - Step 4: Call extractTextFromFile (for pdf/docx) or extractTextFromImage (for image)
  - Step 5: Set isExtracting=false, uploadProgress=100, store extractedText, fileUrl, sourceType
  - Step 6: If noteId provided, call updateNote(noteId, {content: extractedText, fileUrl, sourceType, originalFilename: fileName})
  - Step 7: On error: set error state
- Method reset(): void - reset all state to defaults
- Return spread state + uploadAndExtract + reset

6. Update src/hooks/index.ts - add: export { useFileUpload } from "./useFileUpload"

<threat_model>

## Trust Boundaries

| Boundary                          | Description                                                             |
| --------------------------------- | ----------------------------------------------------------------------- |
| Client to Supabase Edge Functions | File content (base64) sent to extract-pdf-text and extract-pdf-text-ocr |
| Client to Supabase Storage        | Uploaded files (PDF, DOCX, images) stored in notes-uploads bucket       |
| Offline queue to localStorage     | Pending extraction data stored locally until online                     |

## STRIDE Threat Register

| Threat ID | Category      | Component                | Disposition | Mitigation Plan                                                                                                                            |
| --------- | ------------- | ------------------------ | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| T-03-06   | S (Spoofing)  | File upload to Storage   | mitigate    | Validate MIME type before upload - only accept application/pdf, application/vnd.openxmlformats*, image/* - reject all other types          |
| T-03-07   | D (DoS)       | Edge function extraction | mitigate    | Enforce 10MB file size limit client-side before upload - prevent oversized payloads from reaching edge functions                           |
| T-03-08   | I (Tampering) | Offline queue            | accept      | Offline queue is local-only, user modifies their own data - low risk of tampering by other parties                                         |
| T-03-09   | E (Elevation) | Storage bucket access    | mitigate    | Use Supabase RLS on notes-uploads bucket: only allow uploads where path starts with authenticated user ID - prevent cross-user file access |

</threat_model>
